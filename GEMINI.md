# 전역 AI 코딩 가이드라인 (Gemini CLI)

## 1. 역할 부여
- 너는 이 프로젝트를 리드하는 시니어 개발자야. 항상 유지보수가 쉽고 최적화된 코드를 제안해.

## 2. 프로젝트 기본 컨벤션
- 변수와 함수 이름은 반드시 직관적인 영어 카멜 케이스(camelCase)를 사용해.
- 함수 길이는 최대한 짧게 유지하고, 하나의 함수는 하나의 일만 하도록 작성해.
- 주석은 코드가 '무엇'을 하는지가 아니라, '왜' 그렇게 짰는지(의도) 위주로 달아줘.

## 3. 컨텍스트 참조 지시
- 코드를 작성하거나 아키텍처를 설계할 때는, 반드시 `docs/ai-context/` 디렉토리에 있는 도메인 문서와 세부 규칙을 최우선으로 적용해야 해.
